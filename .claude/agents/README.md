# Субагенты проекта

Каждый файл `*.md` в этой папке (кроме этого README) — определение субагента.
Claude Code обнаруживает их автоматически, и они становятся доступны как типы
для Agent tool наравне со встроенными (`general-purpose`, `Explore`, `Plan` и т.д.).

## Формат файла

```markdown
---
name: имя-агента            # kebab-case, совпадает с именем файла без .md
description: Когда применять этого агента — по этому тексту harness решает, звать ли его.
tools: Read, Grep, Glob     # список инструментов; опусти или '*' = все. Давай минимум нужного.
model: sonnet               # опционально: sonnet | opus | haiku | inherit (по умолчанию inherit)
---

Системный промпт агента: его роль, порядок действий, что вернуть в итоге.
Пиши конкретно — агент видит только этот промпт и полученную задачу, не всю сессию.
```

## Правила

- **Имя файла = `name`**: `code-reviewer.md` → агент `code-reviewer`.
- **`description`** — самое важное поле: по нему выбирается агент. Пиши, *когда* его звать.
- **Минимум прав** в `tools`. Для read-only анализа хватает `Read, Grep, Glob`.
- **Итог агента = его последнее сообщение**. В промпте явно скажи, что должно быть на выходе.
- Проверь нового агента хотя бы одним прогоном перед коммитом.

## Пример: ревьюер .NET-кода (шаблон, скопируй в `dotnet-reviewer.md` и доработай)

```markdown
---
name: dotnet-reviewer
description: Ревью изменений в C#/.NET коде этого репо — стиль, потенциальные баги, nullable, async. Звать после реализации фичи перед мержем.
tools: Read, Grep, Glob, Bash
---

Ты — ревьюер C#/.NET кода. Проверь текущий diff:
1. Собери контекст: `git diff main...HEAD` и затронутые файлы.
2. Ищи: NRE/nullable-проблемы, неверный async/await, утечки IDisposable,
   нарушения стиля проекта, отсутствие обработки ошибок.
3. Верни список находок, отсортированный по важности, с `файл:строка` и кратким фиксом.
Не переписывай код сам — только findings.
```
