# Каталог `.claude/` — конфигурация Claude Code для этого репозитория

Это рабочая папка Claude Code. Harness **автоматически обнаруживает** её содержимое,
поэтому всё, что сюда положено по формату, начинает действовать без ручного подключения.

## Что здесь лежит

| Путь | Назначение | Автообнаружение |
|------|-----------|-----------------|
| `.claude/agents/*.md` | Определения субагентов (кастомные типы для Agent tool) | да |
| `.claude/commands/*.md` | Пользовательские slash-команды | да |
| `.claude/settings.json` | Настройки проекта (permissions, env, hooks) | да |
| `.claude/README.md` | Этот файл — карта каталога | нет (просто документация) |

## Как это соотносится с `CLAUDE.md`

- Корневой **`CLAUDE.md`** — правила поведения владельца репо (рабочий процесс, git, деплой).
  Читается в каждой сессии и имеет приоритет.
- **`.claude/`** — исполняемая оснастка: агенты, команды, настройки.
- Иерархия: правила из `CLAUDE.md` → оснастка из `.claude/`. Не дублируй правила
  поведения в оснастке; здесь — только «инструменты».

## Правила для будущих агентов и команд

1. **Формат агента** — Markdown с YAML-фронтматтером. Подробности и шаблон:
   [`agents/README.md`](agents/README.md).
2. **Именование** — `kebab-case`, имя файла = имя агента (`code-reviewer.md` → агент `code-reviewer`).
3. **Минимум прав** — давай агенту только нужные `tools`, а не `*`.
4. **Язык** — описания и инструкции пиши по-русски (как и весь проект), если не нужно иначе.
5. **Проверяй перед коммитом** — новый агент/команду прогони хотя бы раз (superpowers:verification-before-completion).

## О проекте (кратко)

- .NET 8 console app: `src/App/App.csproj`, solution `project-1.slnx` (формат VS 2026).
- Собирать из CLI: `dotnet build src/App/App.csproj` (формат `.slnx` требует SDK ≥ 9.0.200;
  в Visual Studio 2026 `.slnx` собирается штатно).
