# ExamPlanner

Приложение для студентов: экзамены с датами, темы с прогрессом и сложностью,
детерминированное планирование учебного времени, учёт занятий и статистика.

**Платформа MVP:** Android (.NET MAUI), локально, один пользователь, без сети.

## Структура

| Проект | TFM | Назначение |
|--------|-----|-----------|
| `src/ExamPlanner.Core` | `net8.0` | Вся логика: модели, SQLite-репозиторий, сервисы планирования/статистики/бэкапа. Без MAUI, полностью юнит-тестируется. |
| `src/ExamPlanner` | `net10.0-android` | MAUI-приложение (MVVM, CommunityToolkit.Mvvm) поверх Core. |
| `tests/ExamPlanner.Core.Tests` | `net8.0` | xUnit-тесты ядра. |
| `src/App` | `net8.0` | Console-заготовка (не используется). |

## Возможности (Core)

- Экзамены и темы; статус темы (не начата / в процессе / пройдена) и **сложность** (лёгкий/средний/тяжёлый), масштабирующая оценку времени.
- Расчёт темпа, сводка и **дневной план** «Сегодня» по срочности.
- **Доступность по дням недели** + авто-предложение недельного графика с проверкой выполнимости к дедлайнам.
- **Учёт учебного времени** (сессии) и **статистика** (всего / за день / по экзаменам / по дням).
- **Калибровка** оценок по фактически потраченному времени.
- **Бэкап** данных в JSON (экспорт/импорт).

Отложено (с UI/ИИ/сервером): квизы и «двойная оценка», многопользовательский серверный бэкенд.

## Сборка и тесты

`dotnet` вызывать полным путём (не в PATH): `"C:\Program Files\dotnet\dotnet.exe"`.

```bash
# тесты ядра
"C:\Program Files\dotnet\dotnet.exe" test tests/ExamPlanner.Core.Tests

# приложение на Android-эмуляторе
"C:\Program Files\dotnet\dotnet.exe" build src/ExamPlanner -f net10.0-android -t:Run
```

CI (GitHub Actions) прогоняет тесты ядра на каждый push/PR в `main`.

> **Важно:** держать репозиторий на ASCII-пути (напр. `E:\Projects\project-1`) — Android-сборка (AAPT2) падает на путях с не-ASCII символами.
